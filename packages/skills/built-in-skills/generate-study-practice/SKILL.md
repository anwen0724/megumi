# 生成初中学习练习

生成与用户目标匹配、可以实际作答并且经过基本检查的练习，而不是只改写几个数字。

## 确认约束

1. 从当前请求和上下文提取学科、年级或范围、知识点、难度、数量、题型、语言，以及是否立即显示答案和解析。
2. 用户已经给出足够条件时直接出题。只有缺失条件会明显改变结果时才追问；轻微缺口可以采用合理默认并简短说明。
3. 用户要求“类似题”时，保留原题训练的核心知识和推理结构，同时更换情境、数据或设问，避免复制原题。

## 生成与检查

- 让题目覆盖目标知识点，并在用户要求多题时保持适度变化，不把同一道题机械重复。
- 控制语言、计算量和背景知识在相应初中范围内；用户指定挑战难度时可以提高综合性，但不要暗中超纲。
- 逐题检查条件是否充分、答案是否唯一或开放性是否说明清楚，并实际验证计算与参考结论。
- 用户要求先自行作答时，只输出题目，隐藏答案与解析；不要用提示、文件名或排版意外泄露结果。
- 用户要求答案时，将答案与解析放在题目之后的独立区域，并说明关键步骤，而不是只给结果。
- 用户提交作答后，按其要求检查，不因本 Skill 自动生成长期错题记录或能力画像。

## 交付结果

使用清晰编号和适合作答的排版。明确题目范围、必要说明以及答案是否已隐藏。只有用户要求时才保存为文件；未指定路径时将文件放在当前 Workspace 根目录，并在回复中说明产物位置。
