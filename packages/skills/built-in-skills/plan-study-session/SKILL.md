# 安排一次学习过程

把用户当前的学习目标转化为可以马上执行的一次安排，不建立长期课程系统或学情档案。

## 收集必要条件

1. 从请求和会话中确认本次目标、可用时间、资料、截止点和用户已经明确的偏好。
2. 信息足够时直接制定安排。只有缺少目标或时间等关键条件会使计划失真时才追问，不要求填写完整个人资料。
3. 如果用户同时提出多个目标，依据截止时间、依赖关系和实际工作量排序，并说明取舍依据。

## 制定计划

- 把学习过程拆成少量连续阶段，每个阶段写明要做的动作、预计用时、使用的资料和可观察的完成标准。
- 先处理阻碍后续学习的关键理解，再安排练习、检查和必要回顾；不要只按资料页码平均分配时间。
- 预留与总时长相称的检查或缓冲时间。时间很短时主动缩小范围，保留最重要的成果。
- 计划必须允许用户按自己的要求调整讲解、练习或休息方式，不强制固定学习法。
- 如果某项任务需要读取资料、联网或生成文件，正常使用现有工具和权限流程；不要把工具调用写成尚未完成的学习成果。

## 交付结果

用简洁的时间段或顺序清单给出安排，并在结尾列出本次结束时应得到的具体结果。说明计划所依赖的关键假设和超时后的删减顺序。使用用户当前语言，不推断长期掌握度，也不自动保存个人学习记录。
