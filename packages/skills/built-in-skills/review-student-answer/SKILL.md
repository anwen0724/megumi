# 检查学生作答

根据题目要求逐步核对用户的真实作答，保留正确部分并准确定位问题。

## 准备检查

1. 从文字、图片、会话和可用资料中区分题目、评分要求、学生步骤与最终答案。
2. 如果题目或关键作答内容缺失，只询问无法继续检查的具体部分；不要要求用户重新提供已经清楚的信息。
3. 先独立确认一条可靠的解题或判断路径，再与学生作答逐步比较。不要只根据最终答案相同或不同下结论。

## 批改过程

- 明确指出哪些步骤或观点是正确的，并说明它们为什么成立。
- 按实际推理顺序定位第一个实质错误，引用或概括对应步骤，解释错误原因及其对后续结果的影响。
- 区分概念错误、推理错误、计算或书写失误、单位与格式问题，避免笼统地说“方法不对”。
- 如果答案正确，明确确认，并检查过程是否完整、表达是否满足题目要求；不要为了显得有反馈而虚构问题。
- 如果后续步骤只是由前面的错误连带造成，说明这种关系，不把每一步都重复判错。
- 除非用户要求完整重做，否则优先给最小且可执行的修改提示，让用户能够自己修正。

## 交付结果

按“结论—正确部分—首个问题—如何修改”的顺序组织反馈。用户要求分数或等级时，只有存在明确评分标准才据此判断；否则说明只能提供参考评价。使用用户当前语言，不进行人格、能力或长期学情评价。
